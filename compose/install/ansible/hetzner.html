Installation#

Required software#

  • docker
  • bash
  • ssh-keygen
tip

Ansible itself is not required as we run it in docker

  1. Clone the pektin-compose-ansible repo:
  • GitLab
  • GitHub
  • git.y.gy
git clone https://gitlab.com/pektin/pektin-compose-ansible
git clone https://github.com/pektin-dns/pektin-compose-ansible
git clone https://git.y.gy/pektin/pektin-compose-ansible
  1. Create the new file pektin-compose-ansible/playbooks/hcloud.yml and paste the following content into it:
plugin: hcloudtoken: REPLACE_THIS_WITH_YOUR_TOKENkeyed_groups:    - key: labels      prefix: hcloud_labels

  1. Create a new Project in Hetzner Cloud

  2. Go to: your new project > security > api-tokens > create new token > select read-write

  3. Copy the token and paste it into the pektin-compose-ansible/playbooks/hcloud.yml file

  4. Adjust the pektin-config.json

    6.1 Give a name to your public key under ansible.sshPubKeyName

  5. Edit the setup.sh file and replace the value/path of YOUR_SSH_PUB_KEY=/home/paul/.ssh/blackbox.pub with the path to your public key. This will add your public key to all the machines Pektin is getting installed on so that you can ssh into them later.

  6. If this is your first time running the playbook run bash create-keys.sh to create some new SSH keys used for the installation

  7. Run bash setup.sh to start the installation
